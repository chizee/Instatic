# Code Of Conduct

Instatic aims to be a practical, respectful open-source project. Be direct, technical, and considerate.

## Standards

Expected behavior:

- focus criticism on code, design, docs, and tradeoffs
- assume good intent while still being precise about problems
- respect maintainers' time by keeping issues and pull requests focused
- avoid personal attacks, harassment, or discriminatory language

Unacceptable behavior:

- insults, threats, stalking, or sustained disruption
- sexualized language or imagery
- publishing private information without permission
- bad-faith reports, spam, or coordinated abuse

## Enforcement

Maintainers may edit or delete comments, close issues, reject pull requests, block users, or apply other moderation actions when needed to keep the project usable.

For conduct concerns, contact the maintainers through the repository owner's GitHub profile or open a brief issue asking for a private contact path. Do not include sensitive personal details in a public issue.
